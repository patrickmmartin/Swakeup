103 { The ip_option_information structure describes the options to be
104  included in the header of an IP packet. The TTL, TOS, and Flags
105  values are carried in specific fields in the header. The OptionsData
106  bytes are carried in the options area following the standard IP header.
107  With the exception of source route options, this data must be in the
108  format to be transmitted on the wire as specified in RFC 791. A source
109  route option should contain the full route - first hop thru final
110  destination - in the route data. The first hop will be pulled out of the
111  data and the option will be reformatted accordingly. Otherwise, the route
112  option should be formatted as specified in RFC 791. }
